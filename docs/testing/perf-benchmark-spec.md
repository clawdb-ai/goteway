# 性能压测规范（Go Gateway）

日期: 2026-03-17

## 1. 目标

验证 Go Gateway 在保持协议兼容的前提下达到高并发低延迟目标。

## 2. 场景定义

### 场景 A: 连接风暴
- 60 秒内建立 100k WS 连接
- 观测建连成功率、握手延迟、内存峰值

### 场景 B: 请求洪峰
- 10k 并发连接，每连接 5 req/s
- 观测 p50/p95/p99、错误率、CPU

### 场景 C: 事件广播
- presence/tick 高频广播
- 观测慢消费者处理与队列水位

### 场景 D: 流式输出
- 5k 并发 stream 响应（SSE + WS）
- 观测首 token 延迟、流中断率

### 场景 E: 24h Soak
- 持续负载 + 波动流量
- 观测内存泄漏、goroutine 增长、连接抖动

## 3. 指标与门限

- WS 握手成功率 >= 99.99%
- 网关层 p95 <= 10ms
- 网关层 p99 <= 25ms
- 错误率 <= 0.1%（排除上游依赖故障）
- 24h 内存增长稳定在可控阈值内

## 4. 分析方法

- 每场压测保留 profile: cpu/heap/mutex/block
- 输出 alloc/op、ns/op、bytes/op
- 与官方网关同场景对照测试

## 5. 产出

- 压测报告（图 + 数据 + 回归结论）
- 性能回归报警阈值配置
